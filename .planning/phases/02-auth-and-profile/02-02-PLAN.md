Create `backend/src/services/users.service.js`:
- Requires `../config/db` (the pg Pool)
- `getMe(userId)`: `SELECT id, name, email, bio, avatar_url, interests, vibe, vibe_updated_at, created_at FROM users WHERE id = $1` — explicitly excludes password_hash. Returns single row or throws if not found.
- `updateMe(userId, { name, bio, interests })`: Only updates fields that are provided (do NOT overwrite fields omitted from the payload). Build dynamic SET clause: filter `{ name, bio, interests }` for defined values, construct parameterized query. Return updated row (same columns as getMe — no password_hash). If nothing to update, return current profile via getMe.
- `updateAvatar(userId, avatarUrl)`: `UPDATE users SET avatar_url = $1, updated_at = NOW() WHERE id = $2 RETURNING id, avatar_url` — returns updated row.
- Exports: `{ getMe, updateMe, updateAvatar }`

Create `backend/src/controllers/users.controller.js`:
- `getMe(req, res)`: calls `usersService.getMe(req.user.id)`, responds 200 `{ data: user, error: null, message: 'OK' }`. On error: 500.
- `updateMe(req, res)`: extracts `{ name, bio, interests }` from `req.body` (interests may be a JSON string from FormData — parse with `JSON.parse` if it is a string; otherwise use as-is). Calls `usersService.updateMe(req.user.id, { name, bio, interests })`. Responds 200 with updated user. On error: 500.
- `updateAvatar(req, res)`: requires `req.file` (Multer/Cloudinary sets this). If no file, respond 400 `{ data: null, error: 'NO_FILE', message: 'Avatar file required' }`. Calls `usersService.updateAvatar(req.user.id, req.file.path)`. Responds 200 with `{ data: { avatar_url: result.avatar_url }, error: null, message: 'Avatar updated' }`.
- Exports: `{ getMe, updateMe, updateAvatar }`
